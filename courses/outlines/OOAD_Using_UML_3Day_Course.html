OBJECT ORIENTED ANALYSIS AND DESIGN USING THE UNIFIED MODELING LANGUAGE (UML)
3 Days
 
  PREREQUISITE: Students should have some experience with analysis and design.

WHO SHOULD ATTEND?: This course was designed for business or system analysts, technical managers, and software developers who have a need for performing an object oriented analysis and design or anyone who intends to be a course participant in object oriented languages such as C++, or JAVA.

COURSE DESCRIPTION: This course uses the industry-standard Unified Modeling Language as the means of expressing OO concepts, and providing team members with a common notation and vocabulary for communicating their ideas. This course emphasises the conceptual basis of OO through continuous application of key words.

BENEFITS OF ATTENDANCE: Students will be able to:
 

     
       Explain the difference between object and procedural orientation.
       Understand object-oriented concepts such as encapsulation, inheritance, and polymorphism.
       Compare object-oriented analysis with other approaches to systems analysis.
       Describe the evolution of the Unified Modeling Language.
       Use common features of UML diagrams.
       Identify actors and use cases for object-oriented applications.
       Write use case descriptions.
       Apply generalization to use cases.
       Create use case diagrams.
       Identify classes, associations, aggregations, and multiplicity.
       Construct class diagrams at various levels of detail.
       Create sequence and collaboration diagrams.
       Identify object states and substates.
       Draw state diagrams.
      Understand general characteristics of UML processes.
       Describe the object-oriented software life cycle.
       Use basic object-oriented design and project metrics.
